## VMバックアップリストア

### 前提

- 対象: WACで管理しているHyper-Vフェールオーバークラスター環境
- VMディスクはCSV上に配置（例: `C:\ClusterStorage\VolumeX\VMs\`）
- WACは管理/監視コンソール。バックアップ実体はバックアップ製品またはPowerShellで実施する

---

### 事前確認（共通）

```powershell
# クラスター正常性確認
Get-ClusterNode
Get-ClusterGroup
Test-Cluster
```

- Integration Services が有効であること
- Production Checkpoint が設定されていること（AD/SQL/Exchange等はアプリ整合バックアップ設定を確認）

---

### バックアップ手順

#### 推奨: クラスター対応バックアップ製品

- バックアップ対象は「ホスト単位」ではなく「クラスター/クラスターロール単位」で登録
- 全ノードを保護対象に含める
- CSV・VM構成・VHDX・チェックポイント関連を対象にする
- バックアップ方式は Hyper-V VSS ベースのアプリ整合を選択
- ノード単独指定だけだと、ライブマイグレーション後に保護漏れが起きやすい

#### 代替: Export-VM（緊急退避・小規模環境向け）

```powershell
# 所有ノードを確認
Get-ClusterGroup -Name "VM名"

# エクスポート（所有ノード上で実行）
Export-VM -Name "VM名" -Path "D:\VMBackups" -CaptureLiveState CaptureDataConsistentState
```

---

### リストア手順

#### パターン1: 同じパスへ上書き復元

```powershell
$vmName = "APP01"

# 1) 停止
Stop-VM -Name $vmName -Force

# 2) クラスターロール解除
Remove-ClusterGroup -Name "Virtual Machine $vmName" -RemoveResources -Force

# 3) Hyper-V上の登録を削除（ディスクは残す）
Remove-VM -Name $vmName -Force

# 4) vmcxを登録（同一IDで復帰）
Import-VM -Path "C:\ClusterStorage\Volume1\VMs\APP01\Virtual Machines\<vmcx-guid>.vmcx" -Register

# 5) クラスターへ再登録
Add-ClusterVirtualMachineRole -VMName $vmName
```

#### パターン2: 別場所のバックアップから本番パスへ復元

```powershell
$vmName    = "APP01"
$restoreRoot = "D:\Restore\APP01"
$vmPath    = "C:\ClusterStorage\Volume1\VMs\APP01"
$vhdPath   = "C:\ClusterStorage\Volume1\VMs\APP01\Virtual Hard Disks"

Stop-VM -Name $vmName -Force
Remove-ClusterGroup -Name "Virtual Machine $vmName" -RemoveResources -Force
Remove-VM -Name $vmName -Force

Import-VM -Path "$restoreRoot\Virtual Machines\<vmcx-guid>.vmcx" `
    -Copy `
    -VirtualMachinePath $vmPath `
    -VhdDestinationPath $vhdPath

Add-ClusterVirtualMachineRole -VMName $vmName
```

> **注意**
> - 上書き復元では `-GenerateNewId` は使わない（IDを維持して戻す）
> - 先に既存VM登録を削除しないと競合しやすい
> - 実行ユーザーに Hyper-V 管理権限 + クラスター管理権限が必要

---

### 復元後の確認項目

| 確認項目 | 確認方法 |
|---|---|
| VM起動可否 | Start-VM / WAC |
| OSログオン可否 | コンソール接続 |
| アプリ整合性（DB/サービス） | アプリログ確認 |
| クラスターフェールオーバー成功 | Move-ClusterVirtualMachineRole |
| 監視/バックアップ再登録 | 製品コンソール確認 |
| WACでクラスター管理下に戻っていること | WAC画面確認 |

## Pester

```inventory
OsBasic:
  hostname: "test"
  installLang: "0411"
  winPatches:
    - name: kb506139
      state: true
    - name: kb506138
      state: true
```

```pester.ps1
import-module pester
import-module powershell-yaml

$yamlContent = Get-content -raw -path ./inventory.yml -Encoding utf8
$config = ConvertFrom-Yaml $yamlContent

$osBasicCases = @(
  @{
    key = 'hostname'
    expected = $config.OsBasic.hostname
    actualScript = { $env:COMPUTERNAME }
  }
  @{
    key = 'installLang'
    expected = $config.OsBasic.installLang
    actualScript = { ('{0:D4}' -f (Get-WinSystemLocale).LCID) }
  }
)

Describe "sample-test" {
  It "<key> should match (expected: <expected>)" -ForEach $osBasicCases {
    param($key, $expected, $actualScript)

    $actual = & $actualScript

    Write-Verbose "key=[$key] expected=[$expected] actual=[$actual]"
    $actual | Should -Be $expected
  }
}
```

```powershell
Invoke-Pester -Path .\sample.tests.ps1 -Verbose -Output Detailed
```
