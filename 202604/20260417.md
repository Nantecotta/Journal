## Windows2022におけるOS設定項目の検査コマンド

### 前提

- なるべくワンライナーコマンドで目的のパラメタを返り値として取得したい
  - 返り値はパラメタのみ。table形式は不要

### 検査項目

- os基本項目
  - version
    - product-name
      - `Get-CimInstance Win32_OperatingSystem | Select-Object -ExpandProperty Caption`
    - edition
      - `(Get-ItemProperty 'HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion').EditionID`
  - manage
    - admin-password
      - `(Get-LocalUser -Name 'Administrator').PasswordLastSet`
    - インストール言語
      - `(Get-WinSystemLocale).Name`
    - 時刻と通貨の形式
      - `(Get-Culture).Name`
    - 入力形式
      - `(Get-WinDefaultInputMethodOverride).InputTip`
    - キーボード(キーボード レイヤード ドライバ)
      - `(Get-WinUserLanguageList)[0].InputMethodTips[0]`
    - timezone
      - `(Get-TimeZone).Id`
    - リモートデスクトップ有効
      - `(Get-ItemProperty 'HKLM:\SYSTEM\CurrentControlSet\Control\Terminal Server').fDenyTSConnections`
      - NLA有効
        - `(Get-ItemProperty 'HKLM:\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp').UserAuthentication`
  - disks
    - onboot
      - 作成先
        - `Get-Partition | Where-Object DriveLetter | ForEach-Object { $_.DiskNumber }`
      - LUN
        - `Get-Disk | ForEach-Object { "Disk$($_.Number):$($_.Location)" }`
      - drive-letter
        - `Get-Partition | Where-Object DriveLetter | Select-Object -ExpandProperty DriveLetter`
      - size(GB)
        - `Get-Volume | Where-Object DriveLetter | ForEach-Object { [math]::Round($_.Size/1GB,2) }`
  - software
    - ServicePack for ProLiant
      - `(Get-ItemProperty 'HKLM:\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Uninstall\*' | Where-Object DisplayName -match 'Service Pack for ProLiant').DisplayVersion`
    - HPS Reports
      - `(Get-ItemProperty 'HKLM:\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Uninstall\*' | Where-Object DisplayName -match 'HPS Reports').DisplayVersion`
- os追加設定
  - workgroup
    - domain
      - `(Get-CimInstance Win32_ComputerSystem).PartOfDomain`
    - domain-name
      - `(Get-CimInstance Win32_ComputerSystem).Domain`
  - 時刻同期
    - time-server
      - `(Get-ItemProperty 'HKLM:\SYSTEM\CurrentControlSet\Services\W32Time\Parameters').NtpServer`
  - firewall
    - domain-profile
      - `(Get-NetFirewallProfile -Name Domain).Enabled`
    - private-profile
      - `(Get-NetFirewallProfile -Name Private).Enabled`
    - public-profile
      - `(Get-NetFirewallProfile -Name Public).Enabled`
  - windows-update
    - kb506139
      - `[bool](Get-HotFix -Id KB506139 -ErrorAction SilentlyContinue)`
  - network
    - network#1
      - ip
        - `(Get-NetIPAddress -AddressFamily IPv4 | Where-Object { $_.PrefixOrigin -ne 'WellKnown' -and $_.IPAddress -notlike '169.254.*' } | Select-Object -First 1 -ExpandProperty IPAddress)`
      - gateway
        - `(Get-NetRoute -DestinationPrefix '0.0.0.0/0' | Sort-Object RouteMetric,ifMetric | Select-Object -First 1 -ExpandProperty NextHop)`
      - teaming
        - `(Get-NetLbfoTeam -ErrorAction SilentlyContinue).Name`

### windowsAdminCenterについて

- 別物であること
  - Windows Admin Center (WAC) は、Windows Server の「役割と機能」に含まれる機能ではなく、別途インストールする管理ツール。
  - 「機能の設定」に出る項目とは別カテゴリ。

- 主な役割
  - Hyper-V クラスタやサーバーの一元管理をブラウザから実施する管理ゲートウェイ。
  - ノード状態確認、VM操作、イベント確認、更新管理などを集約できる。

- 公式情報
  - 紹介ページ: https://learn.microsoft.com/windows-server/manage/windows-admin-center/overview
  - ダウンロード: https://aka.ms/WACDownload

- 導入の基本手順
  - 公式ページから `WindowsAdminCenter.msi` を取得。
  - 管理用サーバーにインストールし、HTTPS ポートと証明書を設定。
  - ブラウザで `https://<WACのFQDN>:<port>` にアクセスし、管理対象クラスタを追加。

- 高可用化の考え方
  - 本命は WAC ゲートウェイ実体を冗長化すること (active/passive = active/standby)。
  - LB/Proxy は経路冗長化には有効だが、ゲートウェイ実体の冗長化とは別。
  - 推奨は管理専用の2ノードを用意し、クラスタ名/VIP/証明書で公開する構成。

- 管理対象 Hyper-V クラスタ上に WAC VM を置く場合の注意
  - 平常時は運用可能だが、障害時に管理基盤も同時に停止する循環依存リスクがある。
  - Node-A/Node-B は同一ホストに置かず、障害ドメインを分離する。
  - 代替運用経路 (Hyper-V Manager / Failover Cluster Manager / PowerShell) を必ず用意する。
  - パッチ適用はローリングで行い、同時停止を避ける。

- 結論
  - 最も安全なのは、管理対象クラスタ外に WAC を配置し、対象クラスタをリモート管理する方式。
  - クラスタ内配置を採る場合は、冗長化と代替経路の設計を前提にする。

- 構築に必要な設計要素
  - 配置方式
    - クラスタ外配置 / 管理対象クラスタ内配置のどちらにするか。
  - 可用性方式
    - 単体運用 / active-standby (active-passive) のどちらにするか。
  - 名前解決
    - WAC公開名のFQDN、管理対象ノードのFQDN、逆引き要否。
  - 証明書
    - CN/SAN、発行元(CA)、有効期限、更新手順。
  - 認証/権限
    - Kerberos利用可否、運用者ロール、最小権限設計。
  - ネットワーク/FW
    - 利用者 -> WAC、WAC -> 管理対象の到達性と許可ポート。
  - 運用手順
    - バックアップ、パッチ適用、障害時切替、証明書更新、監視項目。
  - 代替運用経路
    - WAC障害時に使う管理手段 (Hyper-V Manager / FCM / PowerShell)。

- 構築時に確定すべきパラメタ
  - WACゲートウェイ情報
    - `WAC_FQDN`: 例 `wac-gw.example.local`
    - `WAC_PORT`: 例 `443` (または `6516`)
    - `WAC_MODE`: `single` / `ha-active-standby`
  - 高可用化情報 (HA時)
    - `WAC_CLUSTER_NAME`: 例 `wac-cluster`
    - `WAC_VIP`: 例 `10.10.10.50`
    - `PRIMARY_NODE` / `SECONDARY_NODE`: 例 `wac-node-a` / `wac-node-b`
  - 証明書情報
    - `CERT_SUBJECT`: 例 `CN=wac-gw.example.local`
    - `CERT_SAN`: 例 `wac-gw.example.local`
    - `CERT_THUMBPRINT`: 実証明書の拇印
    - `CERT_OPTION`: `installed` / `generate`
  - 接続対象情報
    - `TARGET_CLUSTER_FQDN`: 例 `hvcluster01.example.local`
    - `TARGET_NODES`: 例 `hv01,hv02`
    - `MGMT_ACCOUNT`: 例 `EXAMPLE\\svc_wac_admin`
  - 通信要件
    - `CLIENT_TO_WAC_TCP`: 例 `443`
    - `WAC_TO_TARGET_TCP`: 例 `5985,5986`
    - `DNS_REQUIRED`: `true`
  - インストーラ指定値 (サイレント導入時)
    - `SME_PORT=<WAC_PORT>`
    - `SSL_CERTIFICATE_OPTION=<CERT_OPTION>`
    - `SME_THUMBPRINT=<CERT_THUMBPRINT>`
